DAYCARE API SMOKE TESTS

Tujuan:
- Cek cepat API MVP yang dipakai admin app dan daycare app
- Jalankan di GraphQL Playground setelah backend hidup
- Untuk query yang butuh auth, tambahkan header:
  {
    "Authorization": "Bearer <ACCESS_TOKEN>"
  }

query {

CATATAN

- Jalankan urut dari atas supaya ID yang dibutuhkan sudah ada.
- Untuk owner flow, owner harus punya daycare yang sudah dibuat atau disubmit.
- Untuk parent flow, user parent bisa dibuat dengan mutation register biasa sebelum createParent.
- Kalau ada error, catat query mana yang gagal dan pesan GraphQL-nya.
